# Output Contract

Structure the result in this order:

1. Render/call-chain summary
2. Findings grouped by the seven review dimensions
3. Concrete repository-specific code standard
4. Module design principles
5. Priority refactors

Writing rules:

- Keep summaries short
- Spend most of the space on findings with concrete evidence
- Cite file paths and line references for important findings
- Separate current-state observation from recommendation
- When no issue is found in a dimension, say so explicitly
